# Story 3.11 — ARM MRS/MSR (status register transfer)

**Epic:** 03 - CPU — ARM7TDMI Basis

**Doel:** Lezen en schrijven van CPSR/SPSR.

## Zig concepten

### @bitCast tussen packed struct en integer

MRS kopieert de hele CPSR/SPSR naar een gewoon register. Omdat CPSR een packed struct is, moet je casten:

```zig
// MRS: Rd = CPSR (of SPSR)
const use_spsr = (instruction >> 22) & 1 == 1;  // bit 22: 0=CPSR, 1=SPSR
const psr = if (use_spsr) cpu.get_spsr() else cpu.cpsr;
cpu.r[rd] = @bitCast(psr);  // packed struct → u32
```

### Field mask bij MSR

MSR schrijft niet altijd de hele PSR — een 4-bit mask bepaalt welke delen geschreven worden:

```
Bit 19: f — flags veld (bits [31:24]: N, Z, C, V)
Bit 18: s — status veld (bits [23:16]: reserved)
Bit 17: x — extension veld (bits [15:8]: reserved)
Bit 16: c — control veld (bits [7:0]: mode, T, F, I)
```

```zig
const field_mask_bits = (instruction >> 16) & 0xF;

var mask: u32 = 0;
if (field_mask_bits & 0b1000 != 0) mask |= 0xFF000000;  // f: flags
if (field_mask_bits & 0b0100 != 0) mask |= 0x00FF0000;  // s: status
if (field_mask_bits & 0b0010 != 0) mask |= 0x0000FF00;  // x: extension
if (field_mask_bits & 0b0001 != 0) mask |= 0x000000FF;  // c: control

// Schrijf alleen de gemaskeerde bits:
const old: u32 = @bitCast(cpu.cpsr);
const new_val = (old & ~mask) | (value & mask);
cpu.cpsr = @bitCast(new_val);
```

Dit patroon — `(old & ~mask) | (new & mask)` — is een standaard truc:
1. `old & ~mask` — behoud de bits die NIET in het mask zitten
2. `new & mask` — pak alleen de bits die WEL in het mask zitten
3. `|` — combineer ze

### Belangrijk: mode switch bij MSR

Als de control bits (c) geschreven worden en de mode verandert, moet je `switch_mode()` aanroepen om banked registers te swappen.

## Context

**Waarom zijn MRS/MSR nodig?** De CPSR (Current Program Status Register) en SPSR (Saved Program Status Register) zijn speciale registers die niet via gewone data processing instructies bereikbaar zijn. Je kunt niet `MOV R0, CPSR` schrijven — dat bestaat niet in de ARM instructieset. MRS (Move to Register from Status) en MSR (Move to Status from Register) zijn de enige manier om deze registers te lezen en schrijven. Dit is essentieel voor:
- **Mode switches**: de BIOS of een OS-kernel moet de mode bits in CPSR kunnen wijzigen
- **Interrupt beheer**: IRQs aan/uitzetten door de I-bit in CPSR te togglen
- **Exception return**: SPSR teruglezen om de vorige CPU-state te herstellen na een SWI of IRQ

**Waarom field masks?** MSR schrijft niet altijd de hele PSR — een 4-bit mask bepaalt welke 8-bit "velden" geschreven worden. Dit is een veiligheidsmechanisme: als je alleen de condition flags (N, Z, C, V) wilt wijzigen na een berekening, wil je niet per ongeluk de mode bits of de IRQ disable bit overschrijven. De field mask voorkomt dit soort subtiele bugs die catastrofale gevolgen kunnen hebben (verkeerde mode = verkeerde stack pointer = crash).

## Stap-voor-stap

### Stap 1: `src/core/arm.zig` — MRS implementeren (PSR → register)

MRS is de simpelste van de twee: kopieer de hele CPSR of SPSR naar een general purpose register.

```
Instructieformaat MRS:
Bits [31:28] = condition
Bits [27:23] = 00010
Bit  [22]    = R (0=CPSR, 1=SPSR)
Bits [21:16] = 001111
Bits [15:12] = Rd (destination register)
Bits [11:0]  = 000000000000
```

**Waarom is bit [22] de keuze tussen CPSR en SPSR?** Dit is een conventie door de hele ARM instructieset: bit 22 wordt bij meerdere instructies gebruikt om te kiezen tussen de "gewone" en de "gesavede" variant van de PSR. Dit maakt decodering consistent.

```zig
fn mrs(cpu: *Cpu, instruction: u32) void {
    const use_spsr = (instruction >> 22) & 1 == 1;
    const rd = @as(u4, @truncate(instruction >> 12));

    const psr_value: u32 = if (use_spsr) blk: {
        // SPSR van de huidige mode — NIET beschikbaar in User/System mode!
        break :blk @bitCast(cpu.getSpsr());
    } else blk: {
        break :blk @bitCast(cpu.cpsr);
    };

    cpu.r[rd] = psr_value;
}
```

**Waarom `@bitCast`?** De CPSR is een `packed struct(u32)` in onze implementatie. Om het als een gewone `u32` in een register op te slaan, moeten we casten. `@bitCast` doet dit zonder de bits te veranderen — het is puur een type-conversie.

**Valkuil — SPSR in User/System mode:** User en System mode hebben geen SPSR. Als je MRS SPSR probeert in deze modes, is het resultaat UNPREDICTABLE. De veiligste aanpak is om de huidige CPSR terug te geven of een waarschuwing te loggen:

```zig
pub fn getSpsr(self: *const Cpu) PSR {
    const mode = std.meta.intToEnum(Mode, self.cpsr.mode) catch return self.cpsr;
    return switch (mode) {
        .user, .system => self.cpsr,  // geen SPSR beschikbaar
        .fiq => self.spsr[0],
        .irq => self.spsr[1],
        .supervisor => self.spsr[2],
        .abort => self.spsr[3],
        .undefined => self.spsr[4],
    };
}
```

### Stap 2: `src/core/arm.zig` — MSR met register operand

MSR is complexer: het schrijft een waarde naar (delen van) de CPSR of SPSR, met een field mask die bepaalt welke bytes geschreven worden.

```
Instructieformaat MSR (register):
Bits [31:28] = condition
Bits [27:23] = 00010
Bit  [22]    = R (0=CPSR, 1=SPSR)
Bits [21:20] = 10
Bits [19:16] = field mask (f, s, x, c)
Bits [15:12] = 1111
Bits [11:4]  = 00000000
Bits [3:0]   = Rm (bron register)
```

```zig
fn msr(cpu: *Cpu, instruction: u32) void {
    const use_spsr = (instruction >> 22) & 1 == 1;
    const is_immediate = (instruction >> 25) & 1 == 1;  // bit 25: 0=register, 1=immediate

    // De waarde die geschreven moet worden
    const value: u32 = if (is_immediate) blk: {
        // Immediate: 8-bit waarde geroteerd (zelfde als data processing)
        const imm = instruction & 0xFF;
        const rotate = ((instruction >> 8) & 0xF) * 2;
        break :blk std.math.rotr(u32, imm, @as(u5, @truncate(rotate)));
    } else blk: {
        // Register: waarde van Rm
        const rm = @as(u4, @truncate(instruction));
        break :blk cpu.r[rm];
    };

    // Field mask: welke bytes van de PSR mogen geschreven worden
    const field_mask_bits = @as(u4, @truncate(instruction >> 16));
    const mask = buildFieldMask(field_mask_bits);

    if (use_spsr) {
        var spsr: u32 = @bitCast(cpu.getSpsr());
        spsr = (spsr & ~mask) | (value & mask);
        cpu.setSpsr(@bitCast(spsr));
    } else {
        var cpsr: u32 = @bitCast(cpu.cpsr);
        cpsr = (cpsr & ~mask) | (value & mask);
        cpu.cpsr = @bitCast(cpsr);

        // Als de control bits gewijzigd zijn, kan de mode veranderd zijn!
        if (field_mask_bits & 0b0001 != 0) {
            // c-veld is geschreven → check of mode veranderd is
            const new_mode = std.meta.intToEnum(Mode, cpu.cpsr.mode) catch return;
            cpu.switchMode(new_mode);
        }
    }
}
```

**Waarom de `(old & ~mask) | (new & mask)` truc?** Dit is een standaard bit-manipulatie patroon dat selectief bits overschrijft:
1. `old & ~mask` — wis alle bits die in het mask staan (behoud de rest)
2. `new & mask` — pak alleen de bits die in het mask staan (wis de rest)
3. `|` — combineer: de "bewaarde" bits van old + de "nieuwe" bits van new

Voorbeeld met mask = `0xFF000000` (alleen het f-veld):
```
old CPSR = 0x6000001F  (Z=1, C=1, System mode)
new value = 0x80000000  (N=1)

old & ~0xFF000000 = 0x0000001F  (mode bits behouden)
new & 0xFF000000  = 0x80000000  (alleen N flag pakken)
resultaat         = 0x8000001F  (N=1, mode ongewijzigd ✓)
```

### Stap 3: `src/core/arm.zig` — field mask opbouwen

**Waarom 4 aparte bytes?** De ARM ontwerpers verdeelden de 32-bit PSR in 4 logische "velden" van 8 bits elk. Dit geeft fijnmazige controle over welke delen je wijzigt:

```zig
fn buildFieldMask(field_mask_bits: u4) u32 {
    var mask: u32 = 0;

    // Bit 3 (f): flags veld — bits [31:24] — N, Z, C, V flags
    if (field_mask_bits & 0b1000 != 0) mask |= 0xFF000000;

    // Bit 2 (s): status veld — bits [23:16] — gereserveerd op ARM7TDMI
    if (field_mask_bits & 0b0100 != 0) mask |= 0x00FF0000;

    // Bit 1 (x): extension veld — bits [15:8] — gereserveerd op ARM7TDMI
    if (field_mask_bits & 0b0010 != 0) mask |= 0x0000FF00;

    // Bit 0 (c): control veld — bits [7:0] — mode, T, F, I bits
    if (field_mask_bits & 0b0001 != 0) mask |= 0x000000FF;

    return mask;
}
```

**Waarom is het f-veld het meest gebruikt?** De meeste code hoeft alleen de condition flags te wijzigen (bijv. na een berekening in een coprocessor of na een handmatige vergelijking). Met `MSR CPSR_f, R0` schrijf je alleen de flags en laat je de mode, IRQ-status, en andere control bits met rust. Dit is veel veiliger dan de hele CPSR te overschrijven.

**Waarom is het c-veld gevaarlijk?** Het control veld bevat de mode bits (bits [4:0]), de T-bit (ARM/Thumb), en de interrupt disable bits. Als je per ongeluk het c-veld overschrijft met verkeerde waarden, kun je:
- In een ongeldige CPU mode terechtkomen → crash
- Van ARM naar Thumb mode switchen op het verkeerde moment → foute instructie-decodering
- Interrupts uitzetten en vergeten ze weer aan te zetten → game hangt

### Stap 4: `src/core/arm.zig` — MSR met immediate operand

**Waarom heeft MSR een immediate variant?** Het komt vaak voor dat je een bekende constante in de PSR wilt schrijven. Bijvoorbeeld: `MSR CPSR_c, #0x1F` om naar System mode te switchen. Zonder immediate variant zou je eerst een register moeten laden met de gewenste waarde, wat een extra instructie kost.

De immediate waarde gebruikt hetzelfde formaat als data processing: een 8-bit waarde geroteerd over een even aantal bits. Dit is al geïmplementeerd in stap 2 hierboven (de `is_immediate` branch).

```zig
// De immediate berekening (al in stap 2):
const imm = instruction & 0xFF;
const rotate = ((instruction >> 8) & 0xF) * 2;
const value = std.math.rotr(u32, imm, @as(u5, @truncate(rotate)));
```

**Tip:** De immediate variant wordt bijna altijd gebruikt met het c-veld (mode switches) of het f-veld (flags zetten). Het s- en x-veld zijn gereserveerd op de ARM7TDMI, dus schrijven naar die velden is ongebruikelijk.

### Stap 5: `src/core/cpu.zig` — mode switch detectie bij MSR

**Waarom moet je `switchMode()` aanroepen bij een mode change?** Als MSR de control bits wijzigt en daarmee de mode verandert, moeten de banked registers gewisseld worden. De `switchMode()` functie uit story 3.1 doet dit al. Maar je moet het expliciet aanroepen — het schrijven naar `cpu.cpsr.mode` alleen wisselt de banked registers niet automatisch.

```zig
// In de MSR handler, na het schrijven naar CPSR:
if (field_mask_bits & 0b0001 != 0) {
    // Control bits zijn geschreven — mode kan veranderd zijn
    const new_mode = std.meta.intToEnum(Mode, cpu.cpsr.mode) catch {
        std.log.warn("MSR: ongeldige mode {d}", .{cpu.cpsr.mode});
        return;
    };
    cpu.switchMode(new_mode);
}
```

**Valkuil — T-bit wijzigen via MSR:** De ARM specificatie zegt dat je de T-bit (ARM/Thumb) **niet** via MSR mag wijzigen. In de praktijk negeren sommige ARM implementaties een T-bit write via MSR. Als je dit correct wilt emuleren, mask je bit 5 uit het c-veld:

```zig
// Optioneel: bescherm de T-bit tegen wijziging via MSR
if (!use_spsr) {
    // T-bit (bit 5) niet wijzigen via MSR naar CPSR
    const protected_mask = mask & ~@as(u32, 0x20);  // clear bit 5 in mask
    // gebruik protected_mask i.p.v. mask bij CPSR schrijf
}
```

**Tip:** Op de GBA is het wijzigen van de T-bit via MSR een edge case die zelden voorkomt. Begin zonder deze bescherming en voeg het toe als test-ROMs het vereisen.

### Stap 6: testen

```zig
test "MRS leest CPSR correct" {
    var cpu = Cpu.init();
    cpu.cpsr.n = 1;
    cpu.cpsr.z = 1;

    // MRS R0, CPSR
    mrs(&cpu, 0xE10F0000);  // MRS R0, CPSR encoding

    const expected: u32 = @bitCast(cpu.cpsr);
    try std.testing.expectEqual(expected, cpu.r[0]);
}

test "MSR schrijft alleen flags" {
    var cpu = Cpu.init();
    cpu.cpsr.mode = @intFromEnum(Mode.system);

    // MSR CPSR_f, #0x80000000 (zet N flag, laat rest intact)
    // field_mask = 0b1000 (alleen f-veld)
    msr(&cpu, 0xE328F200);  // MSR CPSR_f, #0x80000000 encoding

    try std.testing.expectEqual(@as(u1, 1), cpu.cpsr.n);
    // Mode moet ongewijzigd zijn
    try std.testing.expectEqual(@intFromEnum(Mode.system), cpu.cpsr.mode);
}

test "MSR mode switch triggert register bank swap" {
    var cpu = Cpu.init();
    cpu.r[13] = 0xAAAAAAAA;  // SP in System mode

    // MSR CPSR_c, #0x12 (switch naar IRQ mode)
    // Verwacht: SP wordt gewisseld naar IRQ's SP
    // En de oude SP (0xAAAAAAAA) wordt opgeslagen in banked_sp[system]
}
```
