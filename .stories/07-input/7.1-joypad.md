# Story 7.1 — Joypad input

**Epic:** 07 - Input

**Doel:** GBA knoppen uitleesbaar maken via KEYINPUT register.

## Zig concepten

### Bitmask — elke bit is een knop

KEYINPUT (0x04000130) is een 16-bit register waar elke bit een knop voorstelt. **Active low** betekent: 0 = ingedrukt, 1 = niet ingedrukt.

```zig
// Bit layout van KEYINPUT:
// Bit 0: A
// Bit 1: B
// Bit 2: Select
// Bit 3: Start
// Bit 4: Right
// Bit 5: Left
// Bit 6: Up
// Bit 7: Down
// Bit 8: R
// Bit 9: L

// Of als packed struct:
const KeyInput = packed struct(u16) {
    a: u1,
    b: u1,
    select: u1,
    start: u1,
    right: u1,
    left: u1,
    up: u1,
    down: u1,
    r: u1,
    l: u1,
    _unused: u6,
};
```

### Bit zetten en clearen

Omdat active low betekent "0 = ingedrukt", start je met alle bits op 1 (geen knoppen ingedrukt) en clear je een bit als een knop wordt ingedrukt:

```zig
var keyinput: u16 = 0x03FF;  // alle 10 bits op 1 = niets ingedrukt

// Knop A ingedrukt → clear bit 0
keyinput &= ~(@as(u16, 1) << 0);
// 1 << 0 = 0b0000_0000_0001
// ~ = 0b1111_1111_1110
// keyinput & dat = bit 0 wordt 0

// Knop A losgelaten → set bit 0
keyinput |= (@as(u16, 1) << 0);
// keyinput | 0b0000_0000_0001 = bit 0 wordt 1
```

### Enum voor knoppen

Een enum maakt de code leesbaarder:

```zig
const Button = enum(u4) {
    a = 0,
    b = 1,
    select = 2,
    start = 3,
    right = 4,
    left = 5,
    up = 6,
    down = 7,
    r = 8,
    l = 9,
};

fn press(self: *Input, button: Button) void {
    // Clear bit (active low: 0 = ingedrukt)
    self.keyinput &= ~(@as(u16, 1) << @intFromEnum(button));
}

fn release(self: *Input, button: Button) void {
    // Set bit (active low: 1 = niet ingedrukt)
    self.keyinput |= @as(u16, 1) << @intFromEnum(button);
}
```

### SDL key events omzetten

SDL stuurt keyboard events die je vertaalt naar GBA knoppen:

```zig
// In je SDL event loop:
fn handle_key_event(self: *Input, key: SDL.Keycode, pressed: bool) void {
    const button: ?Button = switch (key) {
        .x => .a,          // X toets = A knop
        .z => .b,          // Z toets = B knop
        .@"return" => .start,
        .backspace => .select,
        .up => .up,
        .down => .down,
        .left => .left,
        .right => .right,
        .a => .l,          // A toets = L shoulder
        .s => .r,          // S toets = R shoulder
        else => null,
    };

    if (button) |btn| {
        if (pressed) {
            self.press(btn);
        } else {
            self.release(btn);
        }
    }
}
```

### Optionals met `if` — het `|btn|` patroon

In Zig is `?Button` een "optional" — het kan een Button zijn of `null`. Je pakt de waarde uit met `if`:

```zig
const maybe_button: ?Button = get_button(key);

if (maybe_button) |btn| {
    // btn is nu een gewone Button, niet meer optional
    self.press(btn);
}
// Als maybe_button null was, wordt dit blok overgeslagen
```

## Context

- KEYINPUT register (0x04000130): 10 bits, elk bit = een knop
- Bits zijn ACTIVE LOW (0 = ingedrukt, 1 = niet ingedrukt) — dit komt voort uit de pull-up weerstanden in de hardware: niet-ingedrukte knoppen geven een hoog signaal (1), ingedrukte knoppen trekken de lijn laag (0)
- KEYINPUT is **read-only** — de hardware zet de bits op basis van de fysieke knoppen, software kan het register niet schrijven
- Knoppen: A, B, Select, Start, Right, Left, Up, Down, R, L
- Er bestaat ook een KEYCNT register (0x04000132) voor keypad interrupts, maar dat is optioneel en wordt door weinig games gebruikt

## Bestanden

- `src/core/io.zig` — KEYINPUT register read
- `src/platform/renderer.zig` of `src/main.zig` — SDL key events → GBA knoppen mapping

## Stap-voor-stap

### Stap 1: `src/core/io.zig` — KEYINPUT register voorbereiden

Het KEYINPUT register leeft in de Io struct als een eenvoudige `u16`:

```zig
// In de Io struct:
keyinput: u16 = 0x03FF, // bits 0-9 op 1 = alle knoppen losgelaten
```

En in de `read16`-switch:
```zig
0x04000130 => self.keyinput,
```

**Waarom is KEYINPUT read-only?** De hardware (fysieke knoppen) bepaalt welke bits aan staan, niet de software. In onze emulator simuleert de platform-laag (SDL) de hardware: die schrijft naar `keyinput`, en de geëmuleerde game leest het via het I/O register. De game zelf kan het register niet schrijven.

**Waarom begint `keyinput` op `0x03FF`?** Dit is de "active low" conventie van de GBA: een bit op 1 betekent dat de knop **niet** is ingedrukt. `0x03FF` = 10 bits op 1 = alle 10 knoppen losgelaten. Dit is de standaard rusttoestand.

**Waarom active low?** Dit is een hardware-elektronicaconventie. De knoppen op de GBA zijn verbonden met pull-up weerstanden. Als een knop niet is ingedrukt, "trekt" de weerstand de spanning omhoog (1). Als je de knop indrukt, wordt de lijn naar ground getrokken (0). De GBA leest deze elektrische signalen direct — vandaar dat 0 = ingedrukt.

### Stap 2: `src/core/input.zig` — Button enum en press/release

Maak een aparte input-module met een duidelijke Button enum:

```zig
pub const Button = enum(u4) {
    a = 0,
    b = 1,
    select = 2,
    start = 3,
    right = 4,
    left = 5,
    up = 6,
    down = 7,
    r = 8,
    l = 9,
};

pub const Input = struct {
    keyinput: *u16,  // pointer naar de u16 in Io struct

    pub fn init(keyinput_ptr: *u16) Input {
        return Input{ .keyinput = keyinput_ptr };
    }

    pub fn press(self: *Input, button: Button) void {
        self.keyinput.* &= ~(@as(u16, 1) << @intFromEnum(button));
    }

    pub fn release(self: *Input, button: Button) void {
        self.keyinput.* |= @as(u16, 1) << @intFromEnum(button);
    }
};
```

**Waarom een pointer naar `keyinput` in Io?** De Input struct leeft in de platform-laag (SDL), maar het register leeft in de core (Io). Met een pointer wijzigt de platform-laag direct de waarde die de game leest — geen kopieën, geen synchronisatie nodig.

**Waarom `enum(u4)` en niet `enum(u16)`?** De enum waarden lopen van 0 tot 9 — dat past in 4 bits. We gebruiken de enum alleen als index voor de shift-operatie, niet als directe bitmask. (Vergelijk met `IrqType` in Story 8.1, die wél als bitmask wordt gebruikt en daarom `enum(u16)` is.)

**Tip:** Vergeet niet dat `press` een bit *cleart* (AND met NOT) en `release` een bit *set* (OR). Dit is het omgekeerde van wat je intuïtief verwacht! Test dit goed: als je per ongeluk press/release verwisselt, lijkt de game "bevroren" (knoppen zijn altijd ingedrukt of nooit ingedrukt).

### Stap 3: `src/platform/renderer.zig` — SDL key events omzetten

Breid de bestaande `processEvents()` functie uit om key events door te sturen naar de Input struct:

```zig
pub fn processEvents(input: *Input) bool {
    var event: c.SDL_Event = undefined;
    while (c.SDL_PollEvent(&event)) {
        switch (event.type) {
            c.SDL_EVENT_QUIT => return false,
            c.SDL_EVENT_KEY_DOWN => {
                if (event.key.key == c.SDLK_ESCAPE) return false;
                handleKeyEvent(input, event.key.key, true);
            },
            c.SDL_EVENT_KEY_UP => {
                handleKeyEvent(input, event.key.key, false);
            },
            else => {},
        }
    }
    return true;
}

fn handleKeyEvent(input: *Input, key: c.SDL_Keycode, pressed: bool) void {
    const button: ?Button = switch (key) {
        c.SDLK_X     => .a,
        c.SDLK_Z     => .b,
        c.SDLK_RETURN => .start,
        c.SDLK_BACKSPACE => .select,
        c.SDLK_UP    => .up,
        c.SDLK_DOWN  => .down,
        c.SDLK_LEFT  => .left,
        c.SDLK_RIGHT => .right,
        c.SDLK_A     => .l,
        c.SDLK_S     => .r,
        else => null,
    };

    if (button) |btn| {
        if (pressed) input.press(btn) else input.release(btn);
    }
}
```

**Waarom X/Z voor A/B?** Dit is een veelgebruikte emulator-conventie. De GBA-knoppen A en B zitten rechts op het apparaat, net als X en Z op een toetsenbord ten opzichte van de pijltjestoetsen. Je kunt dit later configureerbaar maken, maar een goede standaardmapping maakt testen makkelijker.

**Waarom `?Button` (optional)?** Niet elke toets op het toetsenbord correspondeert met een GBA-knop. Door een optional te gebruiken hoef je geen aparte "is dit een geldige toets?" check te doen — het `if (button) |btn|` patroon handelt beide gevallen elegant af. Toetsen zonder mapping worden stilletjes genegeerd.

**Valkuil:** Vergeet niet om `SDL_EVENT_KEY_UP` af te handelen! Als je alleen KEY_DOWN verwerkt, worden knoppen nooit losgelaten en lijkt de game vast te zitten.

### Stap 4: `src/main.zig` — alles aan elkaar knopen

```zig
pub fn main() void {
    var io = Io.init(&ppu, &irq, &timers, &dma);
    var input = Input.init(&io.keyinput);
    var renderer = platform.Renderer.init();
    defer renderer.deinit();

    while (platform.Renderer.processEvents(&input)) {
        // ... emulatie stappen
        renderer.present();
    }
}
```

**Waarom wordt input vóór de loop aangemaakt?** De Input struct bevat een pointer naar `io.keyinput`. Zolang de Io struct leeft, is de pointer geldig. Door Input aan te maken direct na Io, is de eigendomsrelatie duidelijk. Zig's gebrek aan garbage collection betekent dat jij verantwoordelijk bent voor de levensduur van objecten.

**Tip:** Je kunt de input testen zonder een ROM te laden. Start de emulator, druk op toetsen, en log de waarde van `io.keyinput` elke frame. Als je "A" indrukt, moet bit 0 naar 0 gaan (waarde verandert van `0x03FF` naar `0x03FE`). Dit is een goede sanity check voordat je games test.
